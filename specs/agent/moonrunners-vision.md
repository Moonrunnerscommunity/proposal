## The Moonspeaker: Not a Chatbot, a Narrator

The Moonspeaker isn't a chat interface. It's an interactive audiobook.

**You type. It performs.**

You don't talk to the Moonspeaker — you prompt it. It responds with voice, visuals, and atmosphere. Like directing a narrator who has the entire world of Primordia in their head.

### The Experience

```
You type: "Tell me about the Hunter tribe"

The screen shifts. A low, deliberate voice begins:

"The Hunters were the first to smell the change on the wind..."

As the voice speaks, the UI responds:
- A moody illustration fades in
- A map widget highlights Hunter territory
- Stats appear: 1,847 Hunter wolves, 12% of the pack
- A timeline shows key moments in Hunter history

The voice continues, unhurried, while you watch and listen.
```

This is the DCC insight applied: **audio is the breakout medium**. Jeff Hays didn't just read — he performed with sound design, music, character voices. The Moonspeaker does the same, but interactive.

### The Interaction Model

- **Input**: Text (private, works anywhere)
- **Output**: Voice + generative UI (immersive, performative)

You're not having a conversation. You're experiencing a story that responds to your curiosity.

## The Widget System

The Moonspeaker composes experiences in real-time, selecting from a palette of rendering tools. The response isn't text with embeds — the response *is* the composition.

### The Toolkit

| Widget | Purpose |
|--------|---------|
| Voice narration | The primary output — streams with character and mood |
| Ambient audio | Background music, sound effects, atmosphere |
| Image/NFT display | Single, grid, carousel — wolves, scenes, artifacts |
| Video embed | Lore clips, animated moments |
| Contract stats | Floor, holders, volume — live chain data |
| Timeline | Visual history of events |
| Wolf browser | Explore specific NFTs with their journeys |
| Character card | Introduce key figures with stats and story |
| Map view | Primordia geography, tribe territories |
| Quote block | Styled for lore citations vs. historical references |
| Poll/choice | Interactive story beats, community decisions |

### How It Works

The agent's response is a stream of interleaved voice and widget invocations:

```typescript
response: [
  { type: 'ambient', mood: 'mysterious' },
  { type: 'voice', content: 'The summer of the Crimson Moon changed everything...' },
  { type: 'widget', name: 'timeline', props: { event: 'nogard-kidnapping' } },
  { type: 'voice', content: 'Nogard was just a pup when the shadows came...' },
  { type: 'widget', name: 'character-card', props: { character: 'nogard' } },
  { type: 'voice', content: 'No one knows who took him. But the Moonspeakers remember.' },
]
```

The frontend renders sequentially: ambient audio fades in, voice begins, widgets appear as the narration references them.

## The Content Strategy

- **Royal Road web serial**: The core Moonrunners narrative, written for audio
- **Lore documents**: Tribe histories, character bibles, world rules
- **Season rehashes**: Existing lore rewritten with voice and polish
- **Character moments**: Scenes that develop toyable personalities

This is the canon. The Moonspeaker knows it deeply.

### What the Moonspeaker Does (AI-Performed)

- **Narrates** the world you've built with consistent voice and personality
- **Guides** exploration of lore, characters, tribes, history
- **Personalizes** experiences for holders — their wolves' journeys, their tribe's story
- **Creates** shareable moments — styled cards, clips, scenes featuring specific wolves

The AI doesn't generate the world. It performs the world you've created.

When holders share, they're marketing Moonrunners to audiences who've never heard of NFTs.

## The Path to Products

### Phase 1: Content Foundation (Months 1-6)

- Write 50-75 chapters of Moonrunners LitRPG (write for audio)
- Build the lore bible the Moonspeaker will know
- Develop the Moonspeaker voice and personality
- Create the core widget system
- Establish Discord and Patreon infrastructure

### Phase 2: Audience Capture (Months 6-12)

- Launch on Royal Road (follow the Rising Stars playbook)
- Release Moonspeaker experience to holders
- Convert readers to Patreon with advance chapters
- Package into Amazon KU ebook
- Iterate on Moonspeaker based on usage

### Phase 3: Audio Breakout (Months 12-18)

- Commission professional audiobook production (Soundbooth-tier)
- The Moonspeaker voice should align with audiobook narrator
- Launch on Audible while continuing serialization
- Audio is the breakout lever — invest in production quality

### Phase 4: Physical Products (Months 18-24)

- Approach toy manufacturers (Toikido, PMI Kids' World)
- Apply Pudgy separation: consumer marketing without crypto language
- Holder royalties through licensing (OverpassIP model)
- Target Amazon first, then mass retail
